# Deck Planner

Plan the deck before generating slide JSON.

## Step 1. Identify deck type

Choose one:
- Teaching / explainer
- Executive update
- Strategy / proposal

## Step 2. Define the audience promise

Write one sentence:
- "By the end, the audience should understand ..."

This sentence should align with the north star.

## Step 3. Pick the narrative arc

Use `layout-patterns.md` to choose the arc.

## Step 4. Allocate slide budget

For each section, assign a rough count:
- opening
- context
- core explanation
- examples/evidence
- summary/closing

If duration is short, compress aggressively.

## Step 5. Assign slide types intentionally

Use `template-catalog.md` as the guide.
Do not default everything to bullets.

## Step 6. Define one takeaway per slide

Before writing a slide, complete this sentence:
- "This slide exists so the audience remembers ..."

If you cannot complete it clearly, merge or delete the slide.

## Step 7. Review for redundancy

Check for:
- heading + bullet pairs that can merge
- repeated examples
- generic setup slides with no real value
