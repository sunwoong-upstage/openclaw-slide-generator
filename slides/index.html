Core definition

Harness Engineering은 여러 AI와 시스템이 제멋대로 움직이지 않게, 안전하게 함께 일하도록 묶고 조율하는 기술이다.

  • 단일 AI를 잘 만드는 문제를 넘어선다
  • 여러 구성요소가 함께 움직일 때의 안전과 통제가 핵심이다
이 한 문장이 발표 전체의 앵커입니다. 이후 모든 슬라이드는 이 정의를 풀어주는 역할을 합니다.
Why now

AI가 복합 시스템이 되었다

  • 단일 모델보다 여러 요소의 협업이 중요
  • 센서, 모델, 제어 로직이 동시에 움직인다
Risk

현실 세계에서는 실패 비용이 크다

  • 자율주행, 의료, 로봇은 오류 비용이 높다
  • 그래서 안전한 통제가 핵심이 된다
Shift

경쟁력의 중심이 바뀐다

  • AI를 하나 더 만드는 것보다
  • 여러 AI를 믿고 맡길 수 있게 묶는 것이 중요하다
왜 지금 이 개념이 중요한지 세 가지 카드로 압축했습니다.

가장 쉬운 비유로 이해하기

마차의 마구라는 비유가 가장 쉽습니다.

  • 여러 말이 따로 뛰지 않게 잡아준다
  • 힘의 방향을 하나로 모은다
  • 위험할 때 통제권을 유지한다
A
여러 AI를 묶는다
센서, 모델, 소프트웨어가 따로 놀지 않게 한다
B
같은 목표로 정렬한다
서로 충돌하지 않고 같은 방향으로 움직이게 만든다
C
위험할 때 개입 가능하다
중단, 수정, 우회 같은 안전 제어를 가능하게 한다
비유를 통해 개념을 직관으로 바꾸는 슬라이드입니다.
Core 1

규칙

  • 무엇을 해야 하는지 정의
  • 무엇을 절대 하면 안 되는지 명시
  • 행동 범위를 통제
Core 2

모니터링

  • 상태를 실시간으로 관찰
  • 이상 징후를 조기에 발견
  • 문제를 커지기 전에 대응
Core 3

안전장치

  • 위험 시 즉시 중단 가능
  • 인간 개입 통로 확보
  • 안전한 실패 보장
Core 4

투명성

  • 왜 그런 판단을 했는지 설명 가능
  • 원인 추적 가능
  • 신뢰와 검증의 기반

2026년 트렌드를 한눈에 정리합니다.
Use case

자율주행차

  • 카메라, 레이더, 지도 AI를 함께 제어
  • 안전이 최우선인 의사결정 체계 필요
Use case

의료 AI

  • 영상 분석, 기록 요약, 추천을 연결
  • 잘못된 판단을 줄이고 추적 가능해야 함
Use case

산업 로봇

  • 사람과 같은 공간에서 움직임
  • 위험 행동을 즉시 막아야 함
Use case

스마트시티

  • 교통, 에너지, 치안 AI 연결
  • 시스템 간 충돌 없는 운영이 필요
현실 적용처를 빠르게 보여주는 슬라이드입니다.

North star

앞으로의 경쟁력은 AI를 하나 더 만드는 데 있지 않고, 여러 AI를 믿고 맡길 수 있게 묶는 데 있다.

  • 그래서 Harness Engineering은 기능이 아니라 기반 능력이다
  • 이해하는 것에서 끝나지 않고 설명할 수 있어야 진짜 내 것이 된다
이 발표의 메시지를 한 번 더 강하게 박는 슬라이드입니다.
